# Overview

* * *

* * *

The [Economic Research Division](https://research.stlouisfed.org/) of the [Federal Reserve Bank of St. Louis](http://stlouisfed.org/) has enhanced the economic data services it provides by constructing an [API (application programming interface)](https://en.wikipedia.org/wiki/Application_programming_interface), which allows users to create programs that retrieve data from our servers connected to the Internet.

With our [FRED® API](https://fred.stlouisfed.org/docs/api/fred/index.html), users may query our [Federal Reserve Economic Data (FRED®)](https://fred.stlouisfed.org/) and [Archival Federal Reserve Economic Data (ALFRED®)](https://alfred.stlouisfed.org/) databases to retrieve the specific data desired (according to source, release, category and series among other preferences).

Our API accommodates any programming language that can parse [XML](https://en.wikipedia.org/wiki/XML) or [JSON](https://en.wikipedia.org/wiki/JSON) and communicate with our servers using [HTTPS](https://en.wikipedia.org/wiki/HTTPS). The FRED® API is based on the [REST](https://en.wikipedia.org/wiki/Representational_state_transfer) web service architecture. REST leverages familiar web technologies. Like a website, the FRED® API uses [HTTPS](https://en.wikipedia.org/wiki/HTTPS) to receive requests and send responses. Also like a website, the FRED® API uses [URLs](https://en.wikipedia.org/wiki/Uniform_resource_locator) to specify requests. This web service differs from a normal website by sending [XML](https://en.wikipedia.org/wiki/XML) or [JSON](https://en.wikipedia.org/wiki/JSON) instead of [HTML](https://en.wikipedia.org/wiki/HTML). HTML is a visual medium that's not always strictly formatted and flexible enough for arbitrary data structures.
