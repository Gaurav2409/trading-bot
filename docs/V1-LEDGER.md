## 4. Deliberate divergences (decisions already merged — do NOT "restore to original")

Several ◑ rows are partial because the shipped architecture *replaced* the original design. Treat these as settled ADR-backed decisions:

1. **Domain-agent harness replaced the "no-LangGraph" analyst design (V1 T3, T17).** LangGraph is *contained inside one node* behind `ResearchAgentPort` (ADR-0006), canonical run ledger over checkpoints (ADR-0009), `LLMRole` as the only provider boundary (ADR-0007). The MoA `ResearchTradeThesis`/`ConvictionBand`/`SeamProjector` was **superseded, not deferred** — do not rebuild it.
2. **Categorical `EvidencePacket` seam replaced the conviction-band seam (V1 T3).** The shipped `research/seam.py` (closed categorical allowlist + numeric-run rejection) is *stronger* than the planned `SeamProjector`→`HotPathCandidate`. Extend it; don't reintroduce conviction bands.
3. **`corporate_event` is a new profile the original never enumerated (Ont T10–T11).** The nine profiles in the current design (`fundamental`, `technical`, `macro`, `economic_relationship`, `governance`, `liquidity`, `portfolio`, `sentiment`, `market_mechanics`) are the *new* taxonomy — map WS-5's packet builders onto these, not onto the original analyst names.
4. **Account-partitioned models replaced global ones (V1 T4, T9, T10).** `AccountPortfolioSnapshot`+`OwnerPortfolioCut` (not `GlobalPortfolioSnapshot`); kill is account-generation-scoped; reservations are CAS-row-keyed `(account, cas_version)`. WS-8 adds GLOBAL/MARKET/SYMBOL scopes *on top of* the account model.
5. **Live is guarded by `LiveAuthorityReceipt`, not per-order HITL (V1 T28, T36).** The receipt gate + policy releases (ADR-0001) is the shipped live guard; the HITL backend is *additive* interactive approval on top, not a replacement.
6. **Ontology H-tasks are an intentionally reduced kernel (Ont T13, T14).** `challenger.py`/`promotion.py` are the shipped H13/H14 kernels; the deeper identity/event/relationship modules were marked P1-not-executed. WS-4/5/9 *extend* these kernels.
7. **Two-broker, two-market scope is locked.** India Kite + US Alpaca only. IBKR/Angel One and European/XETRA (from the stale `CLAUDE.md`) are **out of V1 scope**.

## 7. Process for the planner

Every workstream in §5 is its own subproject: **brainstorming → design spec (`docs/superpowers/specs/`) → user review → writing-plans (`docs/superpowers/plans/`) → subagent-driven execution → whole-branch review → PR (the user merges).** Work in an isolated worktree. TDD (red→green), one focused commit per task, offline-deterministic tests in the default gate, live/network tests `@pytest.mark.integration` out of the gate. Do not push/PR/merge on assumptions — the user merges PRs.

**Keep the glossary live (domain-modeling).** [`CONTEXT.md`](../CONTEXT.md) at the repo root is the project glossary — a glossary *only*, no implementation detail. As each workstream crystallizes a term, update `CONTEXT.md` inline in the same change (do not batch); challenge fuzzy/overloaded terms against it before coding. Expected new terms: WS-2 — `Claim`/`AdmissionDecision`/`AvailabilityWindow`/origin-cluster; WS-5 — `ReportedFact` vs `DerivedFact`, `InstrumentBeliefState`, exposure vector; WS-7 — FX lot, Rule-115 rate.

**Offer ADRs sparingly.** Existing ADRs are 0001–0010; the next is **0011**. Offer one only when a decision is hard-to-reverse *and* surprising-without-context *and* a real trade-off. Likely candidates: how `fundamental` numeric spans stay categorical at the seam (WS-5); the evidence-ledger immutability/as-of model (WS-2); how a new source's authority tier binds (early-signal onboarding).

**Governing-document index** (the source of truth these workstreams extend): ADRs `docs/adr/0001–0010`; specs `docs/superpowers/specs/` (live-v1 amendment, trading-world ontology, domain-agent architecture, live-source-watcher); the original 52-task plans `docs/superpowers/plans/2026-07-21-*` (SUPERSEDED as executable, retained as detailed reference for the deferred tasks); research `docs/research/{11–14,17,18,19}`.
